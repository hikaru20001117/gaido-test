# リスクカテゴリ定義

## カテゴリ一覧

| カテゴリID | カテゴリ名 | 説明 |
|-----------|----------|------|
| TECH | 技術 | 新技術・未経験領域・インフラ構成・セキュリティに起因するリスク |
| SCHED | スケジュール | 要件定義遅延・承認遅延・リソース不足・依存関係に起因するリスク |
| COST | コスト | スコープ変更・調達コスト変動・為替・予算超過に起因するリスク |
| QUAL | 品質 | テスト不足・仕様不明確・技術負債・パフォーマンス問題に起因するリスク |
| ORG | 体制 | 担当者離脱・顧客担当変更・スキル不足・コミュニケーション断絶に起因するリスク |
| EXT | 外部依存 | サードパーティ・法規制・顧客都合・外部ベンダー遅延に起因するリスク |

---

## カテゴリ別 チェックリスト

提案書からリスクを抽出する際に、以下の観点でスキャンすること。

### TECH: 技術リスク

- [ ] プロジェクト内で初めて使用する技術・フレームワークが含まれているか
- [ ] 外部システムとの連携（API・データ連携）が含まれているか
- [ ] 大規模データ処理・高負荷が想定される機能があるか
- [ ] セキュリティ要件（認証・暗号化・個人情報）が含まれているか
- [ ] クラウドインフラ（AWS/Azure/GCP）の構築・移行が含まれているか
- [ ] レガシーシステムとの接続・データ移行が含まれているか
- [ ] PoC未実施で実現可能性が未確認の機能があるか

### SCHED: スケジュールリスク

- [ ] 顧客側での要件確定・承認が必要なマイルストーンがあるか
- [ ] 外部ベンダー・パートナーの成果物に依存するタスクがあるか
- [ ] 並行して進める別プロジェクトとのリソース競合が想定されるか
- [ ] 法定・制度的なデッドライン（法改正対応・年度末リリース等）があるか
- [ ] キーパーソン（特定スキル保有者）が1名しかいないボトルネックがあるか
- [ ] 前工程（要件定義・設計）の遅延がそのまま後工程に影響する構造か

### COST: コストリスク

- [ ] 要件定義後にスコープが拡大する可能性があるか（追加開発リスク）
- [ ] ハードウェア・ライセンス調達コストが見積もり時点で未確定か
- [ ] 為替変動が調達コストに影響する海外製品・サービスが含まれるか
- [ ] 工数見積もりに大きな不確実性があるタスクが含まれるか
- [ ] 運用コスト（クラウド費用等）がトラフィック量依存で変動するか

### QUAL: 品質リスク

- [ ] 受け入れテストの基準（非機能要件・性能基準）が明確に定義されているか
- [ ] テスト環境の準備（本番相当データ・外部システムスタブ等）が困難か
- [ ] 仕様が曖昧なまま実装を進める箇所が存在するか
- [ ] ユーザビリティ・アクセシビリティ要件が後から追加される可能性があるか
- [ ] 回帰テスト自動化が未整備で、変更の影響範囲確認が属人的か
- [ ] セキュリティ診断・脆弱性検査が必要か

### ORG: 体制リスク

- [ ] プロジェクト期間中に顧客担当者が変わる可能性があるか
- [ ] 自社側で特定技術のスキルを持つ要員が限られているか
- [ ] 下請けベンダーへの依存度が高い（コア部分を外注するか）
- [ ] 顧客側の意思決定ラインが不明確、または意思決定者へのアクセスが難しいか
- [ ] リモートワーク・異なるタイムゾーンによるコミュニケーションコストがあるか
- [ ] 顧客内に IT リテラシーの低い関係者が含まれ、教育コストが発生するか

### EXT: 外部依存リスク

- [ ] 法規制・業界規制の変更によって仕様変更が生じる可能性があるか
- [ ] 外部 API・SaaS（連携先サービス）の仕様変更・廃止リスクがあるか
- [ ] 顧客都合によるプロジェクト中断・スコープ大幅変更のリスクがあるか
- [ ] 調達先ベンダーの納期遅延・供給停止リスクがあるか
- [ ] パートナー企業の経営状況・担当変更リスクがあるか

## 発生確率の目安

| レベル | 確率範囲 | 判断基準 |
|-------|---------|---------|
| 高 | 30% 以上 | 類似プロジェクトで実際に発生した実績がある、または発生要因がすでに顕在化している |
| 中 | 10〜30% | 環境・条件次第で発生しうる。類似案件での発生率が 1〜3 割程度 |
| 低 | 10% 未満 | 発生条件が限定的で、通常の対策で十分に回避できる |

---

## 影響度の見積もり観点

影響度は「このリスクが顕在化した場合、追加で発生するコスト（万円）」として試算する。

### 工数換算の場合

```
影響度（百万円） = 追加工数（人日） × 単価（万円/人日） / 100
```

標準単価の目安（SIer基準）:
- シニアエンジニア: 6〜8 万円/人日
- ミドルエンジニア: 4〜6 万円/人日
- ジュニアエンジニア: 2〜4 万円/人日

例: 追加工数 20 人日 × 5 万円/人日 = 100 万円 = **1.0M**

### 直接費用の場合

再調達費用、ライセンス追加費用、外注費用など実費を直接見積もり、百万円単位で表す。

### 期待損失額の計算

```
期待損失額（¥M） = 発生確率（%） × 影響度（¥M） / 100
```
