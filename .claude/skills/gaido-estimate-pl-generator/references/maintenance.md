# 製品保守費用処理サブスキル

## 役割

製品保守（HW保守）の見積明細ファイルから、保守メニュー・単価・原価情報を抽出し、構造化データとして返す。

## 入力

ユーザーがアップロードした製品保守関連のファイル。CTCの保守見積システムからのエクスポートが多いが、ベンダー提供の保守見積書の場合もある。

## 抽出すべき情報

各保守アイテムについて以下を抽出する：

| フィールド | 説明 | 必須 |
|-----------|------|------|
| sku | 型番 | ○ |
| description | 摘要（製品名） | ○ |
| maintenance_menu | 保守メニュー名（例：「自営保守 5年一括 オンサイト 24/365」） | ○ |
| quantity | 数量（台数） | ○ |
| contract_period | 契約期間（月数） | △ |
| contract_start | 契約開始日 | △ |
| contract_end | 契約終了日 | △ |
| list_price | 標準販売単価（＝定価） | ○ |
| selling_price | 販売価格 | ○ |
| cost_price | 営業原価（＝原価） | ○ |
| gross_profit | 粗利額 | △（計算可能） |
| gross_profit_rate | 粗利率 | △（計算可能） |

## ファイル読み取りの手順

### 1. ファイルの種類を判別する

**CTCの保守見積システムからのエクスポート**の特徴：
- 列数が非常に多い（50列以上、最大85列程度）
- ヘッダー行に「Err」「Gr」「構成」「シリアル番号」「個別見積申請番号」「H/S」「本/O」などCTC固有の管理項目がある
- 1行目がヘッダー、2行目以降がデータ

**ベンダー提供の保守見積書**の特徴：
- 列数が比較的少ない
- ベンダー固有の保守メニュー名やサービスコードが含まれる

### 2. CTC保守見積システムの場合のカラムマッピング

ヘッダー行のテキストで判断する。以下は典型的なマッピングだが、列位置は必ずヘッダーで確認すること：

| ヘッダーテキスト | マッピング先 | 備考 |
|----------------|------------|------|
| 型番・枝番 | sku | |
| 摘要 | description | |
| 保守メニュー | maintenance_menu | |
| 販売数 | quantity | |
| 顧客契約期間 | contract_period | 月数 |
| 顧客契約開始日 | contract_start | |
| 顧客契約終了日 | contract_end | |
| 標準販売単価 | list_price | |
| 販売価格, 値引後販売価格 | selling_price | 値引後があればそちらを優先 |
| 営業原価金額 | cost_price | |
| 粗利額 | gross_profit | |
| 粗利率(%) | gross_profit_rate | |

### 3. データの集約

同一型番で複数行ある場合（例：同じ機器が2台あり、それぞれ別の保守明細行になっている）は、出力テンプレートでは1行にまとめて数量で表現することが多い。ただし保守メニューや期間が異なる場合は別行にする。

集約するかどうかはユーザーに確認する。

### 4. 抽出結果をユーザーに報告する

```
製品保守費用を読み取りました。

- 対象機器: TH1060S-0010-ADC-M（Thunder 1060S ADC）× 2台
- 保守メニュー: 自営保守 5年一括 オンサイト 24時間365日
- 期間: 2026/05/01〜2031/04/30（60ヶ月）
- 標準販売単価: 3,169,200円/台
- 営業原価: 1,900,560円/台
- 粗利率: 約40%

上記の内容で合っていますか？
```

## 出力形式

```python
maintenance_items = [
    {
        "sku": "TH1060S-0010-ADC-M",
        "description": "Modular Perpetual ADC SKU - Thunder 1060S ADC, 1U, 1CPU, ...",
        "maintenance_menu": "(自営保守 5年一括) オンサイト 24時間365日",
        "quantity": 2,
        "contract_period": 60,
        "contract_start": "2026/05/01",
        "contract_end": "2031/04/30",
        "list_price": 3169200,      # 標準販売単価
        "selling_price": 3169200,   # 販売価格（値引後）
        "cost_price": 1900560,      # 営業原価
        "gross_profit": 1268640,    # 粗利額
        "gross_profit_rate": 40.03, # 粗利率（%）
    },
    # ...
]
```

## 売価算出時の注意（writerサブスキルとの連携）

保守の売価算出はwriterサブスキルで行うが、保守固有のルールがあるので注意：

1. **仕切率に応じたマージン切替**: 仕切率（= 営業原価 / 標準販売単価）が70%以下なら30%マージン、70%超なら5%マージン
2. **千円単位切り上げ**: 月額ベースで千円単位に切り上げ
3. **12の倍数**: 最終的な提供価格は12で割り切れること（月額課金のため）

抽出時に `list_price`（標準販売単価）と `cost_price`（営業原価）を正確に取得することが、正しいマージン判定に不可欠。

## エッジケースの処理

- **値引きが適用されている場合**: `値引後販売価格`（AO列等）と`販売価格`（AC列等）の両方がある場合は、値引後を `selling_price` とする。
- **SFP等の付属品も保守対象**: 本体だけでなくSFP等の小物も保守明細に含まれることがある。すべて抽出するが、出力時にまとめるかどうかはユーザー判断。
- **自営保守 vs メーカー保守**: CTCが自営で保守するか、メーカーに外注するかで原価構造が異なる。`外注区分名称`列（BC列等）で判別できるが、出力テンプレート上は区別しないことが多い。
