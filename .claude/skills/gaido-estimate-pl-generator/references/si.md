# SI費用処理サブスキル

## 役割

SI（システムインテグレーション）の役務原価に関する入力ファイルから、作業内容・工数・原価情報を抽出し、構造化データとして返す。

## 入力

ユーザーがアップロードしたSI費用関連のファイル（工数表、SI費用一覧、見積内訳書など）。

## 抽出すべき情報

各SI作業項目について以下を抽出する：

| フィールド | 説明 | 必須 |
|-----------|------|------|
| description | 作業内容の説明 | ○ |
| man_months | 工数（人月） | ○ |
| cost_price | 原価単価（1人月あたり） | ○ |
| cost_total | 原価合計 | ○ |
| deliverables | 成果物リスト（あれば） | △ |

## ファイル読み取りの手順

### 1. ファイルを開いて全体を俯瞰する

Excel / PDF / PPTX いずれかの形式で提供される。形式に応じた読み取りルールは `.claude/rules/constraints.md` に従う:
- Excel: openpyxl（.xlsx）またはxlrd（.xls）で直接読む
- PDF: pdfinfo でページ数確認後、Read tool の pages パラメータで20ページずつ読む
- PPTX: libreoffice --headless --norestore --convert-to pdf で変換後、Read toolで読む

### 2. SI費用の構造を把握する

SI費用の入力ファイルには以下のようなパターンがある：

**パターンA: 工数表形式**
- 行ごとにフェーズ（設計、構築、テスト等）が並ぶ
- 列に「工数（人月）」「単価」「金額」がある

**パターンB: 見積内訳書形式**
- 作業全体がひとまとめで「SI費用一式」として記載
- 内訳として対応内容と成果物が文章で記載される

**パターンC: 担当者別の工数積み上げ**
- 担当者やスキルレベルごとに単価が異なる
- 各フェーズ×担当者のマトリクス形式

### 3. ヘッダー行を探す

以下のキーワードで特定する：

`作業内容`, `フェーズ`, `工程`, `工数`, `人月`, `MM`, `Man-Month`, `単価`, `金額`, `合計`, `SE費`, `役務`, `成果物`

### 4. データを抽出する

作業内容が複数行にまたがる場合（設計・構築・テスト等が別々の行にある場合）は、それぞれ別アイテムとして抽出する。

一方で「SI費用一式」のように1行にまとまっている場合は、descriptionに対応内容と成果物を含めて1アイテムとして抽出する。

### 5. 抽出結果をユーザーに報告する

```
SI費用を読み取りました。

- 作業内容: 設計・構築・テスト（単体、結合、性能）
- 工数: 4人月
- 原価合計: 4,800,000円
- 成果物: NW基本設計書、NW構成図、設定シート、config 等

上記の内容で合っていますか？
```

## 出力形式

```python
si_items = [
    {
        "description": "【対応内容】\n・設計（該当部分を更新）\n・構築\n・テスト（単体、結合、性能）\n\n【成果物】\n・NW基本設計書\n・NW構成図\n・設定シート\n・config\n・監視パラメータ\n・ベンダー試験結果報告書\n・作業手順書",
        "man_months": 4,
        "cost_price": 1200000,    # 1人月あたりの原価
        "cost_total": 4800000,    # 原価合計
        "deliverables": [         # 成果物リスト（あれば）
            "NW基本設計書",
            "NW構成図",
            "設定シート",
            "config",
            "監視パラメータ",
            "ベンダー試験結果報告書",
            "作業手順書"
        ]
    },
    # ...
]
```

## エッジケースの処理

- **原価単価が不明で合計のみ**: `cost_price = cost_total / man_months` で逆算する。
- **工数が人日で記載**: 人月に換算する（1人月 = 20人日が一般的。ユーザーに確認してもよい）。
- **複数フェーズを合算すべきか個別か**: ユーザーに確認する。出力テンプレートのサンプルではSIは1行にまとまっているケースが多い。
