# 運用費用処理サブスキル

## 役割

運用保守・SMS（システムマネジメントサービス）費用の入力ファイルから、初期費用とランニング費用を抽出し、構造化データとして返す。

## 入力

ユーザーがアップロードした運用費用関連のファイル。CTCのSMS見積書、運用保守提案書など。

## 抽出すべき情報

運用費用は「初期費用」と「ランニング費用」の2つに大別される。

### 初期費用
| フィールド | 説明 | 必須 |
|-----------|------|------|
| item_name | 費用項目名 | ○ |
| amount | 金額 | ○ |

### ランニング費用
| フィールド | 説明 | 必須 |
|-----------|------|------|
| item_name | 費用項目名 | ○ |
| monthly_cost | 月額費用 | ○ |
| period_months | 期間（月数） | ○ |
| period_total | 期間合計 | ○ |

### 前提条件・提供役務
| フィールド | 説明 | 必須 |
|-----------|------|------|
| conditions_initial | 初期費用の見積前提条件 | △ |
| conditions_running | ランニング費用の見積前提条件 | △ |
| services_initial | 初期の提供役務 | △ |
| services_running | ランニングの提供役務 | △ |

## ファイル読み取りの手順

### 1. 費用サマリーシートを探す

運用費用ファイルには複数のシートが含まれることが多い。費用の数値データが入っているシートを優先的に探す：

- 「費用サマリ」「SMS費用」「コスト」「見積」等のシート名を探す
- RFI（要件）シートや構成図シートは費用データではないので後回し

### 2. 費用サマリーシートの構造を読み取る

典型的な構造：

```
費用明細 | 項目 | 月額費用 | ヶ月 | 期間合計
---------+------+---------+------+---------
初期費用
  1) SMS開始準備           |   - |   - | 450,000
  2) ITOX開始準備          |   - |   - | 150,000
  初期費用 合計            |     |     | 600,000
SMS費用
  1) インシデント対応
    ・インシデント対応(HW)   | 75,000 | 12 | 900,000
    ・運用関連問い合わせ      | 187,500 | 12 | 2,250,000
  ...
  SMS費用 合計             | 814,125 |    | 10,855,800
```

「初期費用」「ランニング費用（SMS費用）」の境界を見つけ、それぞれの小計行・合計行を特定する。

### 3. ヘッダーと小見出しを探す

以下のキーワードでセクションを判別する：

**初期費用セクション**: `初期費用`, `初期`, `イニシャル`, `Initial`
**ランニングセクション**: `SMS費用`, `ランニング`, `月額`, `運用費用`, `Monthly`, `Running`
**合計行**: `合計`, `小計`, `Total`, `計`

### 4. 前提条件・提供役務シートも確認する

「提供役務」「見積前提条件」等のシートがあれば、そこから前提条件と提供役務のテキストを抽出する。これは見積書の「御見積条件」に反映する際の参考情報となる。

### 5. 抽出結果をユーザーに報告する

```
運用費用を読み取りました。

■ 初期費用（合計: 600,000円）
  - SMS開始準備: 450,000円
  - ITOX開始準備: 150,000円

■ ランニング費用（月額合計: 814,125円、年間合計: 10,855,800円）
  - インシデント対応(HW): 75,000円/月
  - インシデント対応(運用関連): 187,500円/月
  - リモート接続: 75,000円/月
  - インシデント管理・月次レポート: 75,000円/月
  ...

■ 前提条件: 提供役務・見積前提条件シートあり（見積条件に反映できます）

上記の内容で合っていますか？
```

## 出力テンプレートへの配置

運用費用は機器・SI・保守とは性質が異なり、テンプレートへの配置方法はユーザーと相談して決める。オーケストレーターがPhase 2でユーザーに確認する際、以下の選択肢を提示すること：

1. パターンシートに「運用（初期）」「運用（ランニング）」セクションを追加
2. 別シート「運用費用」として独立させる
3. 見積サマリーに含めるがパターンシートには含めない

## エッジケースの処理

- **月額が記載なく年額のみ**: `monthly_cost = period_total / period_months` で逆算する。
- **期間が12ヶ月以外**: 契約期間が36ヶ月等の場合もある。`period_months` に実際の値を入れる。
- **費用サマリーシートが複数パターン**: 24H365Dと平日日勤帯で2パターンある場合、ユーザーにどちらを使うか確認する。
- **対象機器シート**: 運用保守の対象機器リストが含まれることがある。直接的な費用データではないが、機器費用との突合に使える参考情報として抽出しておく。
