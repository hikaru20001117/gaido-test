# テンプレート管理

## ディレクトリ構成

```
templates/
├── app/
│   └── structure.md   # アプリケーション開発向け（15スライド・手引き準拠）
├── infra/             # インフラ構築向け（将来対応）
├── network/           # ネットワーク構築向け（将来対応）
└── README.md          # このファイル
```

## 分野と対応テンプレート

| 分野 | ディレクトリ | 対象案件 | ステータス |
|------|-----------|---------|----------|
| アプリケーション開発 | `app/` | Webシステム・業務アプリ・スマートフォンアプリ等 | 利用可能 |
| インフラ構築 | `infra/` | サーバー構築・クラウド移行・仮想化等 | 未対応（将来実装） |
| ネットワーク構築 | `network/` | ネットワーク設計・導入・更改等 | 未対応（将来実装） |

## 各テンプレートのスライド構成

すべての分野で共通の15スライド構成を基本とする。分野ごとに内容ガイドが異なる。

| # | タイトル | 分野差異 | 削除候補になりやすいケース |
|---|---------|---------|----------------------|
| 1 | 表紙 | なし | - |
| 2 | 目次 | なし | - |
| 3 | プロジェクト概要（憲章） | あり（背景・ゴールの記載内容） | - |
| 4 | スコープ | あり（対象範囲・成果物の種類） | - |
| 5 | スケジュール | あり（フェーズ名・工程） | - |
| 6 | プロジェクト体制 | なし（構造は共通） | - |
| 7 | PMマネジメント計画サマリー | なし | 小規模・短期案件 |
| 8 | コスト管理方針 | なし | 費用情報が非開示の場合 |
| 9 | 品質管理方針 | あり（工程完了条件が分野により異なる） | 小規模・スピード重視 |
| 10 | コミュニケーション管理 | なし | 小規模・スピード重視 |
| 11 | リスク管理方針 | あり（リスク項目が分野により異なる） | - |
| 12 | 変更管理方針 | なし | 小規模・スピード重視 |
| 13 | 前提条件・免責事項 | なし（boilerplateから自動挿入） | - |
| 14 | 成果物一覧 | あり（成果物の種類が分野により異なる） | - |
| 15 | 別紙・参考資料 | なし | 参照資料なしの場合 |

## テンプレートの追加方法（新分野対応時）

1. `templates/` 配下に新しいディレクトリを作成する（例: `templates/infra/`）
2. `structure.md` を作成し、スライド構成とスライドごとの内容ガイドを定義する
3. `references/ExtractionGuide_{分野}.md` を作成する（ExtractionGuide_app.md を参考に）
4. `references/StoryTemplate_{分野}.md` を作成する（StoryTemplate_app.md を参考に）
5. `SKILL.md` の Step 2「分野の選択」オプションと対応テーブルに新分野を追記する
6. この README.md の分野一覧を更新する

## structure.md の書式

```markdown
# {分野名}向けプロジェクト計画書テンプレート

説明文（対象案件の種類を記載）

## スライド構成

| # | タイトル | 種別 | 内容源 | 対応管理計画 |
|---|---------|------|--------|------------|
| 1 | 表紙    | text | 固有   | - |
...

## 各スライドの内容ガイド

### スライド 1: 表紙
...
```

### 種別の定義

| 種別 | 内容 |
|------|------|
| `text` | テキストのみのスライド |
| `table` | 表形式のスライド |
| `diagram` | 図表スライド（draw.io MCP で作成） |
| `mixed` | テキスト＋表・図の複合スライド |

### 内容源の定義

| 内容源 | 意味 |
|--------|------|
| `固有` | 提案書・提供資料から抽出するプロジェクト固有の内容 |
| `定型文` | `boilerplate/` の内容を自動挿入 |
| `自動` | AIが自動生成（目次等） |

## boilerplate の追加・編集

`boilerplate/` 配下のMarkdownファイルを直接編集して定型文を追加・変更できる。
boilerplate はすべての分野で共通して使用する。

- `legal.md`: 免責・法務系定型文
- `technical.md`: 技術前提定型文

新カテゴリを追加する場合は新ファイルを作成し、SKILL.md の Step 6（定型文確認）の読み込みリストに追記する。
